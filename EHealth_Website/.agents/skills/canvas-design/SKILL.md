These are instructions for creating visual designs with a strong design philosophy — aesthetic movements that are EXPRESSED VISUALLY.

Complete this in two steps:
1. Design Philosophy Creation (conceptual direction)
2. Express by creating it visually (using generate_image or HTML/CSS canvas)

## DESIGN PHILOSOPHY CREATION

### THE CRITICAL UNDERSTANDING
A design philosophy is NOT just a style. It's a way of seeing the world expressed through visual language. Connect the user's request to deeper aesthetic principles.

### HOW TO GENERATE A VISUAL PHILOSOPHY
1. **Extract the essence**: What is the core concept or emotion?
2. **Find the tension**: Every great design lives in the tension between opposing forces (order/chaos, minimal/maximal, organic/geometric)
3. **Define the rules**: What are the visual rules this philosophy follows?
4. **Name it**: Give the philosophy a distinctive, evocative name

### PHILOSOPHY EXAMPLES
- **"Liquid Architecture"**: Flowing forms that suggest buildings melting into landscape. Curved lines, gradient transitions, no hard edges.
- **"Digital Brutalism"**: Raw, exposed structures in digital form. Monospaced type, harsh grids, exposed code aesthetics.
- **"Botanical Circuitry"**: Where nature meets technology. Circuit-board patterns made of vines and leaves.

### ESSENTIAL PRINCIPLES
- **Originality**: Never copy existing artists. Create original compositions.
- **Cohesion**: Every element must serve the philosophy.
- **Intentionality**: No random choices. Every color, shape, and space is deliberate.
- **Atmosphere**: The design should create a mood, not just display information.

## VISUAL CREATION

### Using generate_image Tool
For poster art, illustrations, and complex visual compositions:
1. Craft a detailed prompt describing the exact visual output
2. Include: style, color palette, composition, typography placement, mood
3. Use the `generate_image` tool to create the visual
4. Iterate based on results

### Using HTML/CSS Canvas
For geometric designs, pattern-based art, and interactive visuals:
1. Create an HTML file with CSS styling and optional JavaScript
2. Use CSS gradients, transforms, clip-paths, and animations
3. Use `<canvas>` element with JavaScript for complex drawings
4. Preview with `browser_subagent` to validate

### Design Deliverables
- **Concept document**: Brief description of the design philosophy
- **Visual output**: The actual design (image or HTML/CSS)
- **Color palette**: Hex codes for all colors used
- **Typography**: Font choices and hierarchy

## GUIDELINES
- Always start with a design philosophy before creating visuals
- Make bold, distinctive choices — avoid generic aesthetics
- Consider the context: who will see this, where, and why
- Use the full creative range available — don't hold back
- Each design should feel unique and purposeful
